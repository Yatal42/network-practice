## אפשרות 1: מחשב אחד (Localhost) - מומלץ להתחלה! ⭐

### איך זה עובד?

```
┌─────────────────────────────────────────────┐
│         מחשב Windows אחד                     │
│                                             │
│  ┌────────────────────┐                     │
│  │  Jupyter Notebook  │                     │
│  │  (Cell 22)         │                     │
│  │  שולח חבילות ───►  │                     │
│  └────────────────────┘                     │
│             │                               │
│             ▼                               │
│  ┌────────────────────┐                     │
│  │  Loopback         │  ◄─── כאן "המחשב   │
│  │  127.0.0.1        │       השני" הווירטואלי│
│  └────────────────────┘                     │
│             │                               │
│             ▼                               │
│  ┌────────────────────┐                     │
│  │  Wireshark        │                     │
│  │  צופה בחבילות     │                     │
│  └────────────────────┘                     │
└─────────────────────────────────────────────┘
```

### מה קורה פה?

1. **Jupyter שולח חבילה** מ-127.0.0.1 אל 127.0.0.1
2. **Loopback** - זה כמו "מחשב וירטואלי" בתוך המחשב שלך
3. החבילה "נוסעת" דרך Loopback (לא יוצאת לרשת אמיתית)
4. **Wireshark** על אותו מחשב צופה בחבילות

### יתרונות:
- ✅ **קל להתקנה** - הכל על מחשב אחד
- ✅ **אין בעיות רשת** - לא צריך WiFi/כבל
- ✅ **אין בעיות חומת אש** - הכל פנימי
- ✅ **מספיק למטלה** - ברוב המקרים

### חסרונות:
- ❌ לא "רשת אמיתית" (אבל זה בסדר!)

## אפשרות 2: 2 מחשבים אמיתיים - רשת אמיתית 🌐

### איך זה עובד?

```
       מחשב 1 (שולח)              מחשב 2 (מקבל)
    ┌──────────────────┐        ┌──────────────────┐
    │   Windows        │        │   כל מערכת       │
    │                  │        │   (Win/Mac/Linux)│
    │  IP: 192.168.1.10│        │  IP: 192.168.1.20│
    │                  │        │                  │
    │ Jupyter Notebook │        │                  │
    │ (Cell 22)        │        │                  │
    │ שולח חבילות      │        │                  │
    └─────────┬────────┘        └────────┬─────────┘
              │                          │
              │    ┌────────────┐        │
              └───►│   Router   │◄───────┘
                   │   או WiFi  │
                   └────────────┘
                         │
                    רשת LAN אמיתית
                         │
              ┌──────────▼─────────┐
              │  Wireshark על      │
              │  מחשב 2 רואה       │
              │  את החבילות        │
              └────────────────────┘
```

### מה קורה פה?

1. **מחשב 1** מריץ את Jupyter
2. Jupyter שולח חבילות מ-192.168.1.10 אל 192.168.1.20
3. החבילות **יוצאות לרשת האמיתית** דרך כרטיס הרשת
4. עוברות דרך Router/WiFi
5. מגיעות ל**מחשב 2** (192.168.1.20)
6. **Wireshark על מחשב 2** צופה ותופס את החבילות

### יתרונות:
- ✅ **רשת אמיתית** - כמו באינטרנט האמיתי
- ✅ **הדגמה יותר מרשימה**
- ✅ **לומדים על routing אמיתי**

### חסרונות:
- ❌ צריך 2 מחשבים
- ❌ צריך רשת משותפת
- ❌ עלול להיות בעיות חומת אש
- ❌ יותר מסובך

## 📋 הכנה ל-2 מחשבים (אם בוחרים באפשרות 2)

### במחשב 1 (Windows - שולח):

1. מצא את ה-IP שלו:
   ```bash
   ipconfig
   ```
   חפש: IPv4 Address (למשל: 192.168.1.10)

2. התקן הכל (Python, Scapy, Pandas, Jupyter)

3. פתח את המחברת ב-Cursor

### במחשב 2 (מקבל):

1. מצא את ה-IP שלו:
   - Windows: `ipconfig`
   - Mac: `ifconfig`
   - Linux: `ip addr`
   
   חפש IP (למשל: 192.168.1.20)

2. התקן **רק Wireshark** (זה הכל!)

3. פתח Wireshark ובחר את הממשק הנכון (Ethernet/Wi-Fi)

### בדוק חיבור:

**במחשב 1, הקלד:**
```bash
ping 192.168.1.20
```

**אמור להופיע:**
```
Reply from 192.168.1.20: bytes=32 time=2ms TTL=64
Reply from 192.168.1.20: bytes=32 time=1ms TTL=64
```

**אם עובד - אתם מוכנים! ✅**

**אם לא עובד:**
- בדוק ששני המחשבים באותה רשת (אותו WiFi/Router)
- כבה חומת אש זמנית (Windows Firewall)

## 💡 טיפים חשובים ל-2 מחשבים

### 1. כתובות IP צריכות להיות באותו טווח:
- ✅ 192.168.1.10 ← → 192.168.1.20 (טוב!)
- ❌ 192.168.1.10 ← → 10.0.0.5 (לא טוב!)

### 2. בדוק את Subnet Mask:
צריך להיות 255.255.255.0 בשני המחשבים.

### 3. חומת אש:
אם לא עובד, כבה זמנית:
- Windows: Settings → Firewall → Turn off (זמנית!)

### 4. הממשק הנכון ב-Wireshark:
אם לא רואה חבילות, נסי ממשק אחר.
